# SDD Workflow

Use this skill when a task mentions SDD, requisitos, backlog, historia de usuario, criterios de aceite, MVP, or product documentation.

Workflow:

1. Identify the related user story or requirement.
2. Read the exact doc only when the compact project instructions are not enough.
3. Confirm acceptance criteria before implementation.
4. Keep scope limited to the requested MVP behavior.
5. At the end, report which criteria are satisfied and which remain pending.

Primary docs:

- `docs/01-visao-produto.md`
- `docs/02-requisitos.md`
- `docs/03-regras-negocio.md`
- `docs/06-backlog-mvp.md`

Do not invent enterprise features outside the MVP unless the user asks.
