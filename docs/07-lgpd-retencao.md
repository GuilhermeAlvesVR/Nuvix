# LGPD: Retencao e Consentimento

Este documento define uma politica inicial para tratamento de dados pessoais no sistema. Deve ser revisado juridicamente antes de uso publico como politica oficial.

## Dados Tratados

- Dados de identificacao: nome, telefone, email, documento e endereco quando informados.
- Dados operacionais: agendamentos, pagamentos, despesas e historico de uso.
- Dados clinicos: registros de atendimento, queixas, condutas, observacoes e anexos futuros, quando existirem.
- Dados de auditoria: usuario, acao, entidade alterada e data/hora.

## Finalidade

- Organizar o atendimento e a gestao da empresa contratante.
- Manter historico operacional e clinico necessario para continuidade do atendimento.
- Cumprir obrigacoes legais, fiscais, contratuais e regulatórias aplicaveis.
- Proteger a seguranca do sistema e registrar auditoria minima.

## Retencao

- Dados cadastrais e operacionais devem ser mantidos enquanto a empresa usar o sistema ou enquanto houver obrigacao legal/contratual.
- Dados clinicos nao devem ser removidos automaticamente, pois podem ser necessarios para historico de atendimento, defesa legal e obrigacoes profissionais.
- Pacientes podem ser inativados para interromper uso operacional sem apagar historico.
- Exportacao dos dados do paciente deve estar disponivel para atendimento de solicitacoes LGPD.
- Exclusao definitiva ou anonimizacao deve ser analisada caso a caso, considerando obrigacoes legais e risco de perda de historico clinico.

## Consentimento e Transparencia

- O cadastro publico da empresa deve informar que dados serao tratados para prestacao do servico.
- O cadastro de pacientes deve ser acompanhado pela politica da empresa contratante, quando aplicavel.
- Se o produto decidir exigir aceite explicito, o sistema deve registrar data/hora, texto aceito e usuario responsavel.
- O cadastro publico da empresa exige aceite explicito e registra data/hora e versao dos termos no workspace.
- O sistema deve evitar usar dados pessoais para finalidades nao relacionadas a operacao contratada.

## Seguranca

- Acesso aos dados deve ser limitado por perfil de usuario.
- Senhas devem permanecer armazenadas apenas com hash seguro.
- Logs e exportacoes nao devem expor dados sensiveis alem do necessario.
- Mudancas clinicas e exportacoes LGPD devem manter auditoria minima.

## Pendencias Para Politica Oficial

- Validar prazos legais especificos por area de atuacao.
- Definir texto publico de privacidade e termos de uso.
- Definir fluxo formal para pedidos de exclusao, correcao, exportacao e anonimização.
- Definir responsavel pelo tratamento de dados em cada empresa contratante.
